# コンテンツ分類の3層構造

CAREER EDIT では、記事を「カテゴリ」「タグ」「特集」の3つの軸で整理する。それぞれが異なる役割を持ち、重ねて運用することで、読者と編集部の双方が記事にアクセスしやすくなる。

---

## カテゴリ：記事の「種類」を分類する骨格

### 特徴

- 少数精鋭。5〜7 個に絞る
- 全記事に必ず 1 つ付く
- サイトの構造そのもの。URL 設計にも反映される
- 永続的（簡単に増減しない）

### CAREER EDIT のカテゴリ

1. **企業情報**（COMPANY）— 広告代理店、デジタルエージェンシー、事業会社などの企業ページ
2. **職種図鑑**（JOB）— 戦略プランナー、マーケター等の職種解説
3. **キャリアインタビュー**（INTERVIEW）— 実名・顔出し取材記事
4. **現場の声**（VOICES）— 匿名イニシャル取材
5. **編集部記事**（EDITOR'S NOTE）— 編集部・編集長の視点記事
6. **転職者の声**（TRANSITION）— 転職経験者のインタビュー

---

## タグ：記事の「内容」を横断的に紐付ける属性

### 特徴

- 多数あって OK。50〜100 個程度
- 1 記事に複数付与（3〜5 個が目安）
- 記事を書きながら増やしていく
- 読者の「これに関する記事まとめて読みたい」に応える

### タグのグループ（例）

**企業名タグ**
電通 / 博報堂 / サイバーエージェント / 電通デジタル / 博報堂DYデジタル / アクセンチュア / オプト / ADKHD / セプテーニ

**職種タグ**
マーケター / プランナー / クリエイティブディレクター / データアナリスト / コンサルタント / ストラテジスト / コピーライター / アートディレクター

**業界・領域タグ**
総合代理店 / デジタル専業 / 事業会社 / コンサル / スタートアップ / 外資系 / D2C / SaaS

**年代・属性タグ**
20代 / 30代 / 40代 / 第二新卒 / 女性キャリア / 副業

**テーマタグ**
転職体験談 / 年収交渉 / キャリアパス / 独立 / 海外勤務 / 育児両立 / 転職失敗

**手法・スキルタグ**
運用型広告 / SEO / コンテンツマーケ / ブランディング / データ分析

### タグ運用ルール

- 表記揺れを避ける（例：「マーケティング」と「マーケター」を混ぜない）
- 企業名は正式名称を採用（略称は使わない）
- タグ辞書を管理し、新規追加時は編集長がチェック

---

## 特集：編集部が意図して束ねた「企画」

### 特徴

- 期間限定・テーマ限定の編集パッケージ
- 複数のカテゴリ・タグの記事を編集視点で再編成
- 編集メディアの主役。雑誌の特集号に相当
- 特集ごとにオリジナルのリード文・編集部解説を添える

### 初期の特集案

1. **CAREER EDIT 創刊号**（Vol. 001 / 2026.04〜）
   編集長と編集部からのマニフェスト、業界の地図、初期取材記事を収録

2. **30代マーケターの転機**（Vol. 002 / 2026.04〜）
   30代の分岐点で、事業会社・コンサル・独立・海外の道を選んだ7人の物語

3. **総合代理店、再起動の現在地**（Vol. 003 / 2026.05〜）
   電通・博報堂・ADK・大広の現役・元社員・人事責任者の声を集約

### 特集の頻度・運用

- **頻度**：月 1 本ペースを目指す
- **本数**：各特集で 5〜10 本の関連記事を束ねる
- **期限**：連載終了後もアーカイブとして残す

### 特集のサイト上の位置づけ

- グローバルナビに「特集」として独立配置
- トップページに「現在の特集」セクション
- 特集ページは記事一覧とは別デザイン（雑誌の特集号らしく）

## 3層構造の関係性（実例）

1 本の記事は、以下のように多層で紐づく：

```
記事タイトル：「電通デジタルから事業会社 CMO へ。田中 直樹さん（35歳）」

カテゴリ：キャリアインタビュー
タグ：電通デジタル / マーケター / 30代 / CMO / 事業会社 / 代理店出身 / 転職体験談
特集：30代マーケターの転機（Vol. 2）
```

この記事は：

- 「キャリアインタビュー」カテゴリのアーカイブに入る
- 「電通デジタル」タグのページに表示される
- 「マーケター」タグのページにも表示される
- 「30代マーケターの転機」特集にも入る
- 「電通デジタル」企業情報ページから関連記事として表示される

この多層構造により、読者は自分の関心に応じて異なる入口から記事に到達できる。

---

## 検索機能との関係

検索機能は、カテゴリ・タグ・特集の 3 層を横断的に検索できるように設計する。

- **フェーズ 1（立ち上げ時）**：シンプルなキーワード検索
- **フェーズ 2（記事 100 本超）**：カテゴリ・タグでの絞り込み検索
- **フェーズ 3（記事 500 本超）**：AI 検索（自然言語質問への回答）

## よくある失敗パターンと対処法

### 失敗 1：カテゴリとタグが混ざる

「電通デジタル」をカテゴリにしてしまう → カテゴリが企業数だけ増えて破綻。

**対処**：企業名は必ずタグ。カテゴリは記事の種類だけ。

### 失敗 2：タグの粒度がバラバラ

「マーケティング」タグと「マーケター」タグが両方ある。

**対処**：タグ辞書を最初に決めて管理する。

### 失敗 3：特集が 1 回作って終わる

立ち上げ時に勢いで作って、その後フォローアップしない → 廃墟化。

**対処**：特集は最初は 2〜3 本に抑える。月 1 本のペースを継続できる体制でやる。

### 失敗 4：特集が記事の使い回しすぎる

既存記事を束ねただけで特集と称する → 「ただのまとめ」に見える。

**対処**：特集には特集オリジナルのリード文・編集部解説を必ず付ける。
